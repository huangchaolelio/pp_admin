# 功能规范: 各页面数据展示分页

**功能分支**: `003-paginate-list-views`
**创建时间**: 2026-04-23
**状态**: 草稿
**输入**: 用户描述: "对各页面数据展示进行分页"

## 用户场景与测试 *(必填)*

### 用户故事 1 - 视频分类列表分页浏览 (优先级: P1)

管理员在视频分类页面浏览大量视频记录（当前约 295 条），希望每次只加载一页数据，通过翻页控件逐步浏览，而不是一次性加载所有数据导致页面卡顿。

**优先级原因**: 视频分类是数据量最大的页面（295 条），不分页会导致明显的加载性能问题，优先级最高。

**独立测试**: 可通过打开视频分类页面，观察首次加载数据量是否被限制、翻页控件是否正常工作来完整测试并交付价值。

**验收场景**:

1. **给定** 视频分类页面加载时，**当** 页面打开，**那么** 仅显示第一页数据（默认 20 条），并展示当前页/总页数和总记录数
2. **给定** 用户在第 1 页，**当** 点击"下一页"，**那么** 页面切换到第 2 页数据，URL/状态同步更新
3. **给定** 用户修改筛选条件，**当** 条件变更，**那么** 自动回到第 1 页重新查询
4. **给定** 用户调整每页显示条数，**当** 选择 10/20/50，**那么** 当前页重新加载并展示对应数量的记录

---

### 用户故事 2 - 教学提示列表分页浏览 (优先级: P2)

管理员在教学提示页面浏览共 2607 条教学提示，需要分页加载以保证页面可用性。

**优先级原因**: 数据量最大（2607 条），不分页几乎无法使用，但由于是只读展示页面，优先级略低于可编辑的视频分类。

**独立测试**: 可通过打开教学提示页面，验证首次只加载部分数据、翻页正常工作来独立测试。

**验收场景**:

1. **给定** 教学提示页面加载时，**当** 页面打开，**那么** 仅显示第一页数据（默认 20 条）及总记录数
2. **给定** 用户在任意页，**当** 切换动作类型筛选条件，**那么** 自动重置到第 1 页重新查询

---

### 用户故事 3 - 教练/技术标准/知识库版本列表分页 (优先级: P3)

管理员在教练管理、技术标准、知识库版本页面浏览数据时，页面支持分页（当前数据量较小，但为一致性和未来扩展性提供分页能力）。

**优先级原因**: 数据量较小，暂无性能问题，但统一交互体验有价值，优先级较低。

**独立测试**: 可通过各页面验证分页控件存在且功能正常来独立测试。

**验收场景**:

1. **给定** 教练列表数据少于每页上限，**当** 页面加载，**那么** 显示所有数据且分页控件显示正确的总数
2. **给定** 技术标准/知识库版本列表，**当** 数据超过每页上限，**那么** 分页控件正常工作

---

### 边界情况

- 当筛选结果为 0 条时，显示空状态提示，分页控件隐藏或置灰
- 当总数据少于每页条数时，不显示多余的翻页按钮
- 当用户处于第 N 页时修改筛选条件，自动跳回第 1 页
- 当后端接口返回的 total 为 0 时，正确展示"共 0 条"

## 需求 *(必填)*

### 功能需求

- **FR-001**: 视频分类、教学提示页面必须向后端传递分页参数（当前页、每页条数），不得一次性请求全部数据
- **FR-002**: 各列表页面必须展示分页控件，包含：当前页码、总页数（或总记录数）、上/下一页按钮、跳转至指定页
- **FR-003**: 每页默认显示 20 条记录，用户可在 10/20/50 中选择每页条数
- **FR-004**: 筛选条件变更时，系统必须自动将当前页重置为第 1 页后重新查询
- **FR-005**: 教练管理、技术标准、知识库版本列表需添加分页控件，与其他页面保持一致的交互体验
- **FR-006**: 分页控件在数据加载中时应显示加载状态，防止用户重复操作

### 关键实体

- **分页状态**: 当前页码（currentPage）、每页条数（pageSize）、总记录数（total），各页面独立维护
- **分页参数**: 向后端传递的 page/limit 或 skip/limit 参数（依后端接口实际支持的参数名）

## 成功标准 *(必填)*

### 可衡量的结果

- **SC-001**: 视频分类页面首次加载时，网络请求的数据量不超过 50 条（默认每页 20 条），不再一次性拉取全部 295 条
- **SC-002**: 教学提示页面首次加载不超过 50 条，不再一次性拉取全部 2607 条
- **SC-003**: 所有列表页面的分页控件操作响应在用户感知上即时（点击翻页后 2 秒内展示新数据）
- **SC-004**: 修改筛选条件后自动回到第 1 页的行为在 100% 的筛选操作中生效
- **SC-005**: 各页面分页交互一致，用户无需重新学习即可在不同页面使用分页功能

## 假设

- 后端接口（`/api/v1/videos/classifications`、`/api/v1/teaching-tips`）已支持 `limit`/`skip` 或 `page`/`page_size` 等分页查询参数，返回结构包含 `total` 和 `items`（视频分类已确认此结构）
- 教练、技术标准、知识库版本接口当前数据量较小，若后端不支持分页参数，则在前端做展示层分页兼容
- 分页组件复用 Element UI 的 `el-pagination` 组件，无需引入额外依赖
- 不涉及 URL 参数持久化（刷新页面回到第 1 页为可接受行为）
- 诊断页面无列表展示，不在本次分页范围内
