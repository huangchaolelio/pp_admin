## 用户场景与测试 *(必填)*

### 用户故事 1 - 管理员登录与退出 (优先级: P1)

管理员通过账号和密码登录后台管理系统，进入带有中文导航菜单的控制台主页，完成工作后可安全退出登录。

**优先级原因**: 登录鉴权是所有后台功能的入口，没有可用的登录流程，其他功能均无法访问。

**独立测试**: 可以通过访问登录页面、输入账密、验证跳转到控制台主页、再点击退出完整测试，交付「可登录的后台入口」这一核心价值。

**验收场景**:

1. **给定** 管理员拥有有效账号和密码，**当** 在登录页输入正确信息并提交，**那么** 系统验证通过，跳转至中文控制台主页
2. **给定** 管理员输入错误密码，**当** 提交登录表单，**那么** 页面显示中文错误提示，不跳转
3. **给定** 管理员已登录，**当** 点击头部「退出」按钮，**那么** 清除登录状态并跳转回登录页

---

### 用户故事 2 - 中文侧边栏导航与页面路由 (优先级: P1)

管理员登录后，通过左侧中文菜单在不同功能模块之间切换，浏览器地址栏同步变化，页面内容随之更新，刷新页面不丢失当前位置。

**优先级原因**: 导航是后台所有功能模块的骨架，菜单能正常切换才能为后续业务模块奠定基础，且中文界面是本项目硬性要求。

**独立测试**: 可通过观察侧边栏是否显示中文标签、点击菜单项是否正确切换内容区域来独立验证，与具体业务数据无关。

**验收场景**:

1. **给定** 管理员已登录，**当** 查看左侧导航菜单，**那么** 所有菜单项、子菜单标签均以中文显示
2. **给定** 管理员点击某个中文菜单项，**当** 路由跳转完成，**那么** 内容区域切换到对应页面，面包屑导航同步更新
3. **给定** 管理员在某个子页面刷新浏览器，**当** 页面重新加载，**那么** 仍然停留在同一页面，不重定向到首页

---

### 用户故事 3 - 用户管理基础功能 (优先级: P2)

管理员可以在「用户管理」模块查看系统用户列表，并能新增、编辑、禁用用户账号。

**优先级原因**: 用户管理是后台最核心的基础业务功能，是后续乒乓球教练业务配置的前提（需要管理员账号体系）。

**独立测试**: 可以通过新增一个测试用户、在列表中看到该用户、再编辑其信息来完整验证，与其他业务模块互相独立。

**验收场景**:

1. **给定** 管理员进入「用户管理」页面，**当** 页面加载完成，**那么** 以分页表格形式展示用户列表，列头为中文
2. **给定** 管理员点击「新增用户」按钮，**当** 填写用户名、密码、角色并提交，**那么** 新用户出现在列表中
3. **给定** 管理员选中某用户并点击「禁用」，**当** 操作确认，**那么** 该用户状态变为「禁用」，禁用后该账号无法登录

---

### 用户故事 4 - 全局界面汉化 (优先级: P1)

系统所有界面元素（按钮、表单标签、提示信息、日期选择器、分页控件、对话框等）均显示中文，不出现英文残留。

**优先级原因**: 这是用户输入中明确的硬性要求，影响所有页面，必须在项目初期统一处理，否则后续修复成本高。

**独立测试**: 可通过浏览器逐页巡检所有界面元素语言来验证，不依赖任何业务数据。

**验收场景**:

1. **给定** 管理员访问任意后台页面，**当** 检查所有可见文本，**那么** 按钮、标签、提示、占位符均为中文
2. **给定** 管理员使用日期选择器或下拉组件，**当** 组件展开，**那么** 内置文案（如「确定」「取消」「上一月」「下一月」）均为中文
3. **给定** 系统出现表单验证错误，**当** 错误信息显示，**那么** 提示内容为中文

---

### 边界情况

- 当管理员 Token 过期时，访问受保护页面应自动跳转到登录页并提示「登录已过期，请重新登录」
- 当管理员直接通过 URL 访问未授权页面时，系统应返回中文「无权访问」提示页面
- 当网络异常导致接口请求失败时，页面应显示中文错误提示而非空白或英文报错

## 需求 *(必填)*

### 功能需求

- **FR-001**: 系统必须提供中文登录页面，支持账号密码方式登录，登录成功后跳转控制台主页
- **FR-002**: 系统必须在登录成功后将访问凭证存储于 `localStorage`，有效期 2 小时，每次操作自动续期；凭证过期时自动跳转登录页
- **FR-003**: 系统必须提供带有折叠功能的中文侧边栏导航，支持多级菜单结构；v1 阶段包含「控制台」「用户管理」「乒乓球教练管理（占位）」「系统配置」4 个一级模块
- **FR-004**: 系统必须根据当前用户角色动态渲染可见菜单，隐藏无权限的菜单项
- **FR-005**: 系统的所有 UI 组件（按钮、表单、分页、日期选择器、弹窗等）必须完整汉化
- **FR-006**: 系统必须提供用户管理页面，支持列表查看、新增、编辑、禁用用户操作
- **FR-007**: 系统必须在顶部导航栏展示当前登录用户名，并提供退出登录入口
- **FR-008**: 系统必须支持浏览器刷新后保持当前页面路由，不丢失导航状态
- **FR-009**: 系统必须对所有需要认证的页面进行路由守卫拦截，未登录访问自动跳转登录页
- **FR-010**: 系统必须提供控制台概览首页，展示基础统计数据（用户数、模块数等占位信息）

### 关键实体

- **管理员账号**: 系统的操作主体。必填字段：用户名（全局唯一）、密码、角色；可选字段：手机号、备注。通过用户名+密码登录
- **角色**: 定义权限边界，关联菜单可见性。超级管理员可见全部菜单；普通管理员隐藏「用户管理」和「系统配置」类高权限模块
- **菜单项**: 侧边栏导航的基本单元，具备中文标题、图标、路由路径、所需角色等属性，支持父子层级

## 成功标准 *(必填)*

### 可衡量的结果

- **SC-001**: 管理员可以在 1 分钟内完成登录并进入功能页面
- **SC-002**: 后台所有可见页面文本中，中文覆盖率达到 100%，不出现任何英文残留（组件内置文案、按钮、提示语等）
- **SC-003**: 侧边栏菜单支持至少 2 级嵌套，点击任意菜单项后，内容区域在 1 秒内完成切换
- **SC-004**: 用户管理模块支持对至少 100 条用户记录进行分页展示，每页切换响应流畅无卡顿
- **SC-005**: 90% 以上的测试操作（登录、导航、用户管理）在无需阅读任何文档的情况下可自助完成

## 假设

- 目标使用者为内部运营管理人员，具备基本电脑操作能力，无需针对移动端适配（v1 阶段桌面端优先）
- 基于 vue-element-admin 现有模板，进行中文适配和裁剪，不引入 vue-element-admin 之外的新 UI 框架
- 后台管理系统与 char_pp 主项目通过 API 通信，本功能仅关注前端框架搭建，不涉及具体业务 API 对接
- 初始管理员账号通过配置文件或数据库种子数据预置，不需要注册功能
- 通过 `.env.development` / `.env.production` 环境变量切换 `baseURL`，开发阶段启用 mock，生产环境切换真实 API，接口路径保持一致，业务代码无需修改
- 角色权限在 v1 阶段仅控制菜单可见性，不做操作级细粒度权限控制
